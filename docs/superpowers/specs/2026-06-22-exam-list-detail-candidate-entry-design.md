# 考试列表、详情与考生管理入口优化设计

## 目标

考试列表按考试任务聚合，每个 `taskId` 只显示一行。考试任务详情同时展示正式考试和试考的完整场次信息，并从每个场次提供考生管理入口。现有考试创建、科目创建、试卷绑定、名单整理、考生导入、自动分班和监考账号导出业务逻辑保持不变。

## 范围与保护边界

本次只修改原有考试配置相关页面和前端入口：

- 考试列表 `/exams`
- 考试任务详情 `/exams/:taskId`
- 考生名单整理与导入 `/candidate-import`
- 上述页面需要的前端渲染、路由参数读取和页面边界测试

不得修改需求中心的页面、路由、API、数据库和测试。共享文件中的需求中心导入、DOM 区块、渲染函数、事件处理和 API 分发必须保持原样。

## 考试列表

`/api/exams` 继续返回场次数组，前端使用 `taskId` 建立聚合记录，不新增或修改后端接口。每条聚合记录包含任务名称、来源账号、正式考试场次、试考场次和汇总状态。

列表每个任务只渲染一行，列结构为：

- 考试名称：使用 `projectName`，名称可点击并进入 `/exams/:taskId`
- 正式考试：显示场次名称、`session_id` 和时间
- 试考：显示场次名称、`session_id` 和时间
- 来源账号
- 考生数：分别显示正式考试与试考人数
- 班级数：分别显示正式考试与试考班级数
- 状态：显示任务下场次的汇总状态
- 操作：保留查看和刷新入口，不新增业务操作

搜索同时匹配考试名称、场次名称、`session_id` 和来源账号。现有“场次类型”筛选不再适用于任务聚合列表，因此移除。统计区显示考试任务数、正式场次数和试考场次数。

聚合状态按严重程度计算：任一场次失败则为失败；否则任一场次进行中则为进行中；否则任一场次待人工确认则为待人工确认；全部已有场次成功才为已完成；没有场次或其余情况为未开始。

## 考试详情

详情继续调用 `/api/tasks/:taskId`，不改变任务详情数据结构。页面保留任务摘要和九个固定进度步骤：

1. 需求单解析
2. 正式场次创建
3. 试考场次创建
4. 科目创建
5. 试卷绑定
6. 试考考生导入
7. 正式考试考生导入
8. 试考、正式考试自动分班
9. 试考、正式考试监考账号导出

场次区域采用方案 C：正式考试和试考各一张操作卡。每张卡显示场次名称、`session_id`、考试开始与结束时间、考生数、班级数和状态。两个场次同时存在时并排展示；窄屏时纵向排列。尚未创建的场次显示占位状态，且不提供考生管理按钮。

每个有效场次卡提供“管理考生”按钮。点击后进入：

```text
/candidate-import?taskId=<taskId>&sessionId=<session_id>
```

详情页不复制名单上传、解析、导入或分班控件。

## 考生管理入口

`CandidateImportPage` 进入页面时读取 `taskId` 和 `sessionId` 查询参数：

1. 存在 `taskId` 时调用现有 `/api/tasks/:taskId`。
2. 从任务详情中提取正式考试和试考场次，写入现有场次选择状态。
3. 存在有效 `sessionId` 时默认选中该场次。
4. `sessionId` 无效或缺失时保留两个场次选项，但不自动选中。
5. 不存在 `taskId` 时保持当前独立考生管理页行为，继续通过现有按钮加载租户场次。

后续上传名单、字段映射、校验、模板下载、考生导入、自动分班和监考账号导出全部调用现有函数与接口。任务入口只负责预装场次上下文，不改变任何租户 API 请求体或执行顺序。

## 错误与空状态

- 考试列表接口失败：列表显示请求错误，刷新按钮仍可用。
- 单个任务只有一个场次：另一场次显示“尚未创建”，聚合行仍只显示一次。
- 详情任务不存在：保留详情页框架并显示加载失败信息，不跳转到其他业务页。
- 任务场次没有 `session_id`：显示场次信息，但禁用“管理考生”。
- 考生页任务加载失败：显示错误日志并保留现有手动加载场次入口。
- 查询参数中的 `sessionId` 不属于该任务：不自动选择任何场次，禁止直接导入。

## 测试与验收

自动测试覆盖：

- 同一 `taskId` 的正式考试和试考只渲染一个考试列表行
- 两个不同 `taskId` 始终渲染两行，即使考试名称相同
- 考试名称链接目标为 `/exams/:taskId`
- 搜索可命中任务名、任一场次名、任一 `session_id` 和来源账号
- 详情同时渲染正式考试和试考的七项必要信息
- 九个进度步骤及顺序保持不变
- 两个“管理考生”入口携带正确的 `taskId` 和 `sessionId`
- 考生页从任务详情加载两个场次并正确预选目标场次
- 无效 `sessionId` 不会触发导入
- 需求中心列表、详情 DOM、路由和页面模块仍然存在且未被考试页面引用

本地验收包括访问 `/exams`、打开一个 `/exams/:taskId`、分别从正式考试和试考进入考生管理、刷新三个 URL，并确认浏览器前进/后退行为正常。所有既有 Node 和 Python 测试通过后，再同步到实际部署目录并重启 LaunchAgent。
