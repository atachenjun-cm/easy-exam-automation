# 项目与考试任务详情设计

## 目标

每份需求单对应一个项目。项目包含正式考试与试考两个场次，并保存九个互相独立的任务步骤。项目管理与考试列表展示本机自动化工具创建过的全部记录，不按当前登录账号或租户 API Key 过滤。

## 数据模型

- `exam_tasks` 保存项目名称、来源账号、整体状态、进度、创建时间与更新时间。
- `exam_sessions` 保存任务下的正式考试和试考场次，包括 `session_id`、名称、时间、人数、班级数和状态。
- `exam_task_steps` 保存固定九个步骤的独立状态、时间、耗时、错误、重试次数、子状态和日志。

步骤状态允许独立更新。组合步骤 `sessions_auto_rooms` 与 `sessions_invigilator_export` 仅在正式考试和试考子状态均成功时记为成功。

## 接入方式

- 导入需求单后创建任务，并将 `requirement_parse` 标记为成功。
- 创建正式场次、试考场次、科目和绑定试卷时分别更新对应步骤。
- 现有名单导入、自动分班和监考账号导出逻辑保持不变，只在成功或失败位置调用统一任务步骤更新接口。
- 前端新增项目管理、考试列表和任务详情三个视图，通过后端 API 读取 SQLite。

## 页面

- 左侧导航新增“项目管理”“考试列表”。
- 项目管理展示一份需求单对应的项目卡片和整体进度。
- 考试列表展示所有账号创建的正式考试与试考。
- 任务详情展示项目摘要、场次列表和九张独立步骤卡片；卡片可展开日志，失败步骤可单独重试。

## 验证

- SQLite 单元测试覆盖初始化、跨账号列表、独立步骤更新、组合步骤汇总和刷新读取。
- 后端语法检查与 API 冒烟测试。
- 本地浏览器验证导航、列表、详情与刷新持久化。
